[Tech] Let's make Graphviz support light and dark theme color variations

The origin

I have always been a heavy user of HackMD, whether it’s taking notes in class or recording self-study knowledge, it’s basically all documented on HackMD.

The reason why I like hackMD so much is because it combines many convenient features: for example, you can use `$$ to enclose and type $\LaTeX$ math equations, and use ```graphviz``` to type out flowcharts (Graphviz). This kind of design that only requires text to produce the desired graphics/text, I really think it’s very user friendly.

After planning to start building a personal blog, because it also uses the markdown format for writing, I don’t want to deviate from these two useful functions. The blog theme I use is based on Fluid’s modified version, which also provides built-in light and dark mode. So I found an issue that I didn’t like very much when I looked at it.

Problem

Let’s first take a look at a mathematical expression written in $\LaTeX$:

$$
\sum\limits_{i=0}^{n}a_i
$$

While $\LaTeX$ can support adjusting the font color based on the background color (you can try manually switching between light and dark mode to see it in action), Graphviz does not support this. Because the native hexo does not support rendering Graphviz in the same way as writing in hackMD, I use the package hexo-filter-viz. Let’s take a look at an SVG rendered using the original hexo-filter-viz:



The first problem I noticed it’s not centered! And it looks fine in light mode, but switching to dark mode is very painful. That abrupt background, and the lines that don’t change with the theme color, really feel like they need to be fixed 😢. After a series of adjustments, I finally got the following result:


It looks great! Except for the parts originally in color, the rest will change with the light dark theme, and there will be no background either! Inside the markdown file, I can just use the usual way of using graphviz to generate flowcharts that support dark mode! Now let me introduce what I have done!

Start working

Center

But actually, the first thing that caught my eye was that the default result is not centered, which made me very uncomfortable. So let me manually add the attribute style="text-align:center;" inside the <div id="graphvizID"></div> tag!

Remove the background and change the line color.

Another one is the background color. Although you can manually write bgcolor=transparent; in markdown, I think it’s not elegant enough. So I plan to directly modify the code injected into {%　raw　%} here. First, let’s take a look at what the rendered SVG structure looks like.

Although I only listed a few important components here, we can see from the above rendering result: the outermost svg tag adjusts the overall size of the SVG image, and entering the first g layer represents the entire image. The first layer polygon inside it is the background, and the following g layers represent the components in the image, such as the node representing the shape, and the edge representing the arrow, etc. Inside, there are several smaller elements to form the required shapes.

Here is the translated content:

So here I handle the background and other polygons together:”

Adjust size

Finally, after adjusting, I found that if the given image is too large, it will actually exceed the layout of the entire note (exceeding the right side of the layout). Therefore, I manually adjusted the width to 100% in line 5, so it won’t exceed the layout!

Finished product

Okay! After the above adjustments, there shouldn’t be any major issues after going through one round. So I saved the modified results in hexo-filter-viz-customized. This way, I can use this package in my CI/CD on GitHub to download and properly render Graphviz for me! Awesome!

Conclusion

Actually, I had been thinking about making this modification since the summer of 2023, but I was too busy to do it until winter vacation. I suddenly had the urge to solve this little problem. Originally, when I was in my second year and had not yet completed the software design experiment, I was still avoiding JavaScript on web pages. However, I think that after this implementation, I have found that I am beginning to have the ability to complete the functions I need by reading documents. I think this also counts as a kind of growth XD. Although what I did is very simple, it has significantly improved the overall aesthetics. I look forward to continuing to strengthen and make this blog more complete next time 😎.
